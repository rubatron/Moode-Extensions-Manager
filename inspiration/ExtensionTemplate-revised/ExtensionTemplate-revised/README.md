# Template Extension

Generated by ext-mgr Import Wizard template kit.

## Import behavior
- Installs into /var/www/extensions/installed/template-extension
- Creates canonical route /template-extension.php
- Starts visible in M menu and Library menu, hidden in System/Configure
- Starts with settings-card mode enabled

## Template structure
- ExtensionTemplate/assets
- ExtensionTemplate/backend
- ExtensionTemplate/templates
- ExtensionTemplate/scripts
- ExtensionTemplate/packages
- ExtensionTemplate/data
- ExtensionTemplate/cache

## Default maintenance scripts
- scripts/install.sh is non-destructive by default and only installs the service when permissions allow it
- scripts/repair.sh restores runtime directories under the managed extension root
- scripts/uninstall.sh removes runtime artifacts and disables the extension service when possible

## Packages and dependencies
- Put bundled dependency artifacts under packages/
- packages/services can ship extra systemd units that ext-mgr normalizes to moode-extmgrusr:moode-extmgr
- ext_mgr.install.packages can declare apt packages for host installation
- ext_mgr.service.dependencies can declare unit dependencies to inject into the main extension service

## Logging layout
- Extension global logs: /var/www/extensions/sys/logs/extensionslogs/template-extension
- Extension local logs: /var/www/extensions/installed/template-extension/logs
- Default files: install.log, system.log, error.log

## Staged install hooks
- Optional packages: manifest.json -> ext_mgr.install.packages
- Optional post-copy script: manifest.json -> ext_mgr.install.script
- ext-mgr runs the install script under moode-extmgrusr
- Write files inside /var/www/extensions/installed/template-extension; ext-mgr relocates legacy /var/www/extensions/template-extension writes

## Service parent dependency
- Template manifest includes ext_mgr.service.name=template-extension.service
- Generated service unit requires and starts after moode-extmgr.service
- Keep extension daemons under moode-extmgrusr for consistent permissions

## moOde shell requirements
- Extension pages should include /var/www/header.php and footer integration
- This guarantees Back arrow, Home button and M menu are available
- The generated template.php already follows this requirement

## Menu visibility
- Edit manifest.json -> ext_mgr.menuVisibility
- Current defaults:
  1) m=true
  2) library=true
  3) system=false

## Icon support
- info.json now includes iconClass
- Use the starter icon picker in template.php to pick a class
- Copy the chosen class to info.json -> iconClass
