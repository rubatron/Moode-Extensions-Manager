# Ronnie Pickering's Extension

YouTube video embed extension for moOde Audio Player.
Serves as the reference implementation and import-test package for the ext-mgr system.

---

## Quick start — dev kit

```
delivery/
├── ext-mgr.code-workspace        ← open this in VSCode
├── RonniePickeringExtension/     ← this extension
│   ├── .vscode/                  ← workspace config (tasks, debug, settings)
│   └── scripts/build-zip.sh     ← builds the importable zip

```

Open `ext-mgr.code-workspace` to get all folders in one VSCode window with
pre-configured tasks, debug configs and recommended extensions.

## Building an importable zip

```bash
bash scripts/build-zip.sh              # → ../ronnie-pickering-extension-1.0.0.zip
bash scripts/build-zip.sh -o /tmp      # custom output dir
bash scripts/build-zip.sh --dry-run    # list included files without building
```

Or via VSCode: `Ctrl+Shift+B` → **ext-mgr: Build zip**

The script:
1. Runs `ext_helper.py scan` — aborts if path violations are found
2. Lists all files via Python exclusion logic
3. Packages a clean zip named `<ext-id>-<version>.zip`

**Always excluded from the zip:**

| Excluded | Reason |
|----------|--------|
| `.vscode/` | Dev tooling — not extension content |
| `scripts/build-zip.sh` | Dev tooling |
| `*.code-workspace` | Dev tooling |
| `.gitkeep`, `.gitignore`, `.git/` | VCS artefacts |
| `__pycache__/`, `*.pyc` | Python cache |
| `logs/`, `cache/` | Generated at install time |
| `data/install-footprint.json` | Generated by install.sh |
| `.DS_Store`, `Thumbs.db` | OS junk |

## moOde shell behaviour

- **moOde runtime** (`/var/www/header.php` present): moOde renders its nav shell
  (Library / Audio / Network / … / M-menu). Extension renders only its own
  content block using `ext-template-header`. Branded title hidden.
- **Standalone / dev** (`header.php` absent): full HTML with `<body data-standalone="true">`,
  Google Fonts + Font Awesome from CDN, branded `ext-rp-page-title` shown.

## Sandbox architecture

Everything lives in the extension root. External references point *back* into it.

```
/var/www/extensions/installed/ronnie-pickering-extension/
├── packages/
│   ├── services/    ──symlink──→  /etc/systemd/system/
│   ├── config/      ──symlink──→  /etc/ (udev, lirc, modules…)
│   ├── bin/         ──symlink──→  /usr/local/bin/
│   └── …
└── data/install-footprint.json   ← tracks every symlink + package written outside
```

`/boot/config.txt` is append-only with `# BEGIN <ext-id>` / `# END <ext-id>` guards.
Any `/var/www/` path outside the managed root is a **violation** — `install.sh` aborts.

## File structure

```
template.php                    moOde shell-aware main page
manifest.json                   ext-mgr config (id, type, service, logging, install)
info.json                       name, version, author, iconClass
ext-mgr.menu-preset.json        menu visibility preset
assets/
  css/template.css              glass design, moOde --rp-* token alignment
  js/template.js                copy-to-clipboard, card animations
  images/                       icon.ico / icon.png / icon.svg (custom icon)
backend/
  api.php                       starter JSON endpoint
  ext_helper.py                 Python scanner, rewriter, register, icon handler
scripts/
  install.sh                    intelligent installer — path policy + footprint
  uninstall.sh                  footprint-based surgical uninstaller
  repair.sh                     symlink repair + service health check
  service-runner.sh             daemon heartbeat
  ronnie-pickering-extension.service  systemd main unit
  build-zip.sh                  DEV ONLY — builds importable zip
packages/
  services/                     helper .service units
  data/                         bundled config artifacts
data/                           runtime state (generated)
logs/                           install.log, system.log, error.log (generated)
cache/                          runtime cache (generated)
```

## ext_helper.py CLI

```bash
python3 backend/ext_helper.py scan       .                       # full audit
python3 backend/ext_helper.py footprint  .                       # show footprint
python3 backend/ext_helper.py register   <reg.json> <id> <pkg>   # register apt pkg
python3 backend/ext_helper.py unregister <reg.json> <id>         # release ownership
python3 backend/ext_helper.py rewrite    . <old_id> <new_id>     # rename ID
python3 backend/ext_helper.py save-icon  . <src_file>            # install custom icon
python3 backend/ext_helper.py policy                             # print path policy table
```
