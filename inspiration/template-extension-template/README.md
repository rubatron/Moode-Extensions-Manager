# Template Extension

Generated by ext-mgr Import Wizard template kit.

## Quick Start

1. Replace `template-extension` with your extension ID throughout all files
2. Update `info.json` with your extension metadata
3. Update `manifest.json` with your extension configuration
4. Customize `template.php` with your extension UI
5. Add your JavaScript to `assets/js/template.js`
6. Add your styles to `assets/css/template.css`
7. Implement `scripts/install.sh`, `scripts/repair.sh`, `scripts/uninstall.sh`
8. Zip and import via ext-mgr Import Wizard

## Import Behavior

- Installs into `/var/www/extensions/installed/<id>`
- Creates canonical route `/<id>.php` symlink
- Visible in moOde M menu and Library menu by default

## Manifest Configuration

The `manifest.json` file configures ext-mgr behavior:

```json
{
  "id": "template-extension",
  "name": "Template Extension",
  "version": "0.1.0",
  "main": "template.php",
  "ext_mgr": {
    "enabled": true,
    "state": "active",
    "menuVisibility": {
      "m": true,        // Show in M menu
      "library": true,  // Show in Library menu
      "header": true,   // Header tabs visible on extension page
      "system": false   // Show in System menu (Configure modal)
    },
    "settingsCardOnly": false,
    "headerVisible": true,
    "service": {
      "name": "",           // Service name, e.g., "ext-myservice.service"
      "dependencies": []    // APT package dependencies
    }
  }
}
```

## Header Visibility

The template includes dynamic header visibility control. When enabled, the moOde
navigation tabs are visible on your extension page. This is controlled by
`headerVisible` or `menuVisibility.header` in the registry.

## Services

If your extension includes a systemd service, use the naming convention:
- Service name: `ext-<id>.service`
- Example: `ext-template-extension.service`

Services run under `User=moode-extmgrusr` and `Group=moode-extmgr` for security.
